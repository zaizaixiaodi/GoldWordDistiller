# 句式骨架蒸馏参考 — Abstract Patterns Reference

> **定位**：这不是硬性枚举，而是**蒸馏时的抽象度参照**。
>
> - 蒸馏句式时，**优先**把骨架抽象到下表这种程度（用 A/B/N/X/Y 占位变量，不留具体内容）
> - 如果实际帖子里出现下表覆盖不到的新句式风格，照常提取并新增 — 不要为了套模板而扭曲
> - 这份参考的来源是 `workspace/distilled/abstract_skeletons.json`（上一轮蒸馏 496 条帖子的副产物），代表当前观察到的主流自媒体爆款句式
>
> **核心要求**：以前蒸馏产出的句式（飞书句式库）过于具体（如"女生下班后狠下心来死磕这10种技能"被原样录入），缺乏抽象度，导致无法复用。今后的蒸馏务必先抽象再录入。

|---|---------|------|------|--------------|

| 4 | 触发爆发句：一旦 A，B 就藏不住 | 召唤 | 女生一旦开窍，黑马属性藏不住 / 下班后狠下心来死磕 | who, do, feel |
| 5 | 命令号召句：请/一定要 A | 召唤 | 未来三年普通人一定要小心 / 真心建议所有女生都去探索自由职业 | who, when, do |

## 抽象度判定标准

提取的 `skeleton` 字段应满足：

- ✅ **变量位用占位符** —— A、B、N、X、Y、Z（不要写具体词如"女生""副业"）
- ✅ **保留连接结构** —— 中文连词 / 标点 / 比例符号（→、的、是、从…到…）
- ✅ **可被多个不同内容例句套用** —— 一个 skeleton 至少能匹配 2 条本批帖子，才算"抽象到位"
- ❌ **避免**：把原标题原样塞进 skeleton（如"女生下班后狠下心来死磕这10种技能"）
- ❌ **避免**：粒度过细的子类型（如同时存在"A 岁的我，B"和"A 岁我，B"）

## 输出格式提醒

蒸馏 prompt 输出的 `patterns[]` 应当：

```json
{
  "skeleton": "从 A 状态到 B 状态",        // 抽象骨架，占位符
  "category": "对比",                       // 句式类型（清单/对比/悬念/递进/召唤/承诺/代入/反问 之一）
  "examples": ["从月薪2k到存下上海首付", "从0到第一个100w"],  // 真实例句，最多 3 条
  "from_post_ids": ["recvjHJK...", "recABC..."]
}
```

`examples` 字段保留原帖标题的具体形式，作为这个抽象骨架的"实例证据"。

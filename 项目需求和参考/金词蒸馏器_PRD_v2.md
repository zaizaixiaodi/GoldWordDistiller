# 金词蒸馏器 / GoldWord Distiller — PRD v2.0

> **版本说明**：本文档取代 [`金词蒸馏器_PRD_v1.md`](金词蒸馏器_PRD_v1.md)（已废弃）。v2 在 v1 的"管道"基础上重写"产物"层——金词不再是扁平的关键词清单，而是按功能位 MECE 分类、按来源字段（标题/封面）分别打标、配以独立句式库、并通过周报做元认知反思的三层产物体系。
>
> **产品定位**：一个运行在 Claude Code 下的 CLI Agent，帮助自媒体博主从小红书热贴中蒸馏出可"按位组装"的金词与句式，为选题/标题/封面文案提供原料和模板。
>
> **用户画像**：AI 领域自媒体博主，口播/vlog 形式，覆盖搞钱、职场、个人成长、AI 应用、自媒体方法论等赛道。日常需要持续积累有画面感、力量感、传播力的"金词 + 句式"来驱动选题与文案生产。
>
> **核心价值**：把博主"每天刷帖 30 分钟记热词"的手工劳动，变成"agent 定时采集 + 按功能位结构化蒸馏 + 周报人审精炼"的三层流水线。AI 写手可直接按"选句式 → 按位填金词"模式组装出新标题/封面。

---

## 1. 三层词汇模型（与 v1 一致）

金词蒸馏器的输入侧仍是三层渐进的词汇体系：

**领域词（Domain Words）**：最抽象的一层，描述博主的内容赛道，如"搞钱""职场""个人成长""AI 应用""自媒体方法论"。数量约 5 个，由博主手动设定，长期稳定。

**搜索词（Search Words）**：每个领域词下挂载的具体搜索关键词，如"搞钱"下挂"副业""自由职业""理财"。每个领域词下约 3 个搜索词，总计约 15 个。由博主手动维护，可随热点调整。

**金词（Gold Words）**：从搜索结果中蒸馏出的高价值表达。在 v2 中，金词不再只是"一个有画面感的词"，而是带有以下结构化属性的对象：
- **功能位**（who/when/pain/do/twist/number/feel/picture）—— 见 §4
- **来源字段**（title / cover）—— 同一个词可能在两种场景都出现，但分别打标
- **vibe_score**（画面感/反差感/情绪感的综合得分）

---

## 2. 三层产物模型（v2 核心增量）

v1 只产出"金词清单 + 蒸馏简报"。v2 将产物分为三层：

| 层 | 产物 | 频率 | 受众 | 作用 |
|---|------|------|------|------|
| **L1 原料层** | 金词库（标题/封面 × 8 功能位） | 每次 /harvest | AI 写手 RAG | 海量原料，广撒网 |
| **L2 骨架层** | 句式库 | 每次 /harvest | AI 写手 RAG | 复用句式模板 |
| **L3 精炼层** | 周报（趋势 + TOP金词 + 新句式 + 元认知反思） | 每周 | 人 + 回写 AI | 人审判断，沉淀业务规范 |

**为什么分三层**：
- 没有 L3，L1 会堆成垃圾场——所有词都进来，没人判断哪些过气、哪些组合更高互动
- 只有 L3，AI 没有可调的原料，写手要靠人脑回忆
- L2 独立的原因：句式是"骨架"，金词是"肉"。写文案时 AI 是"选句式 → 按位填金词"。两者绑死会丧失复用性

**AI 写手的使用流程示意**：
```
[领域=搞钱] → 选句式骨架（如"不是 X，是 Y"）
            → 按功能位填词
              · 身份位：独居女生
              · 痛点位：早点下班
              → 输出："不是想躺平，是早点下班的独居女生太累了"
```

## 3. 四个数据入口（与 v1 一致）

### 3.1 平台直出（Auto Harvest）

**触发方式**：定时任务（cron），每周两次（周二 + 周六），也可手动触发。

**流程**：
1. 遍历所有搜索词（约 15 个），逐一调用 TikHub `search_notes`（App V2），按"最多点赞"排序，每个搜索词取 1 页（约 20 条）。
2. TikHub 返回表层数据：标题、帖子链接、封面图 URL、点赞/收藏/评论数、note_type。Agent 侧自行调用图文详情接口补全完整正文（图文笔记），视频笔记不调详情。
3. **封面图采集**：从 `images_list[cover_image_index]` 取博主上传的真实封面 URL，下载到本地，通过 lark-cli 上传到飞书附件字段（详见 DEVLOG 2026-05-16 踩坑）。
4. 写入飞书"热贴库"（含封面附件、URL、互动数据）。
5. **飞书豆包识图**：用户已在飞书多维表上配置豆包多模态识图字段，自动对封面附件 OCR + 语义识别，结果回填到"封面文案输出结果"字段。**飞书架构在 v2 中保持不变**——已验证识图成本可接受，进度优先。
6. Agent 读取已识图的封面文案 + 完整正文 + 标题，进入蒸馏流程（§5）。

**API 调用量估算**：15 次搜索 + 图文笔记详情补充 ≈ 30 次/周，约 120 次/月，≈ $1-2/月（含详情接口）。豆包识图费用由飞书侧计费。

### 3.2 他山之石（Manual Feed）

**触发方式**：`/feed <url>` / `/feed --text` / `/feed --file <path>`。

**流程**：与 v1 一致——用户喂入链接/文本/文件，agent 走相同的蒸馏流程，结果写入金词库（source=他山之石）。

### 3.3 官方榜单（Hot List）

**触发方式**：随 /harvest 一起执行。

**流程**：与 v1 一致——拉热榜，与现有金词库交叉比对，新出现的标记为候选金词。

### 3.4 真人甄选（Human Spark）

**触发方式**：`/add "<金词>" --field <title|cover> --category <功能位> --domain <领域> --note <备注>`

相比 v1，新增 `--field` 和 `--category` 参数，便于人工录入时直接打标。如果不指定，agent 会推断。

## 4. 金词功能位分类（v2 核心增量）

### 4.1 8 个功能位（MECE 切分）

按"金词在标题/封面钩子里承担的功能"切分：

| # | 功能位 (英文锚) | 含义 | 典型例子 |
|---|----------------|------|---------|
| 1 | **身份标签 (who)** | 圈定受众，让读者"对号入座" | 独居女生、高认知穷人、INFJ、35+ 中女、新手、学生党、普通人 |
| 2 | **场景锚点 (when/where)** | 提供时间/空间锚点 | 下班后、凌晨三点、35 岁前、出租屋、高考后的暑假 |
| 3 | **痛点/欲望 (pain/want)** | 读者怕什么/想要什么 | 早点下班、攒下第一桶金、不想内卷、经济独立 |
| 4 | **行动召唤 (do)** | 命令式动词、动作建议 | 把手弄脏、立刻删掉、闭嘴去做、摆摊、起号 |
| 5 | **反常识钩子 (twist)** | 反预期、反直觉的对比/结论 | 越懒越赚、月薪 3 千比 3 万自由、3 个月赚 15w 但离职 |
| 6 | **量化锚 (number)** | 具体数字、时长、金额、等级 | 21 天、月入 5 万、第 7 次、36 种、月薪 6k 存 10w |
| 7 | **情绪/状态词 (feel)** | 情绪/心理状态/网感词 | 班味、松弛感、破防、上头、狠狠、心酸、王炸 |
| 8 | **隐喻具象 (picture)** | 用具体意象代替抽象概念 | 把人生卷成纸团、地毯起号、野路子、摸鱼、卡皮巴拉打工日记、钱生钱 |

### 4.2 必填位 vs 加分位（待回测）

从 §10.1 的样本回测看，搞钱赛道呈现明显的"必填 + 加分"二分：

- **疑似必填位**（出现率 ≥ 35%）：量化锚、身份标签
- **疑似加分位**（出现率 15-25%，但出现时金词味更浓）：反常识、隐喻、情绪
- **疑似辅助位**（出现率 < 20%）：场景、痛点、行动

**待回测**：这一分级仅基于 23 条副业+AI 数据，覆盖单一赛道。Phase 2 蒸馏跑起来后，需用 ≥200 条多赛道数据回测，再决定是否将分级落入 prompt 与简报模板。

### 4.3 第 9 类"工具/IP 名"待回测

回测中出现"DeepSeek+即梦+剪映""卡皮巴拉""Part-time"等词，硬塞进"隐喻具象"略勉强。在搞钱/AI 赛道，"工具堆叠 / IP 借势"可能是独立维度。**Phase 2 跑通后专项回测**，目前先记入 §11 待定事项。

### 4.4 vibe_score（金词味）

每个金词新增一个 `vibe_score` 字段（0-10 整数）：
- 反常识 / 隐喻 / 情绪 三类天然得分高（6-10）
- 量化 / 身份 / 场景 等"信息型"金词得分中等（3-6）
- 通用动词/名词不归为金词，得分 < 3 直接过滤

vibe_score 由蒸馏 prompt 中的 LLM 判断给出，并在周报中作为"TOP 推荐"的排序依据。

## 5. 句式库（v2 新增第 4 张表）

### 5.1 为什么独立成表

句式是金词的"骨架"。同一个句式可以容纳不同金词，复用价值极高。如果与金词混在一张表，会丧失"按句式调用"的检索路径。

### 5.2 常见句式骨架（初稿）

| 类型 | 骨架 | 示例 |
|------|------|------|
| 对比 | "宁愿 X 也不 Y" / "与其 X 不如 Y" / "X 比 Y 更香" | 宁愿少赚一点也不卷 |
| 递进 | "不是 X，是 Y" / "X 不是错，Y 才是" | 不是想躺平，是太累了 |
| 悬念 | "这件事我不敢早说" / "看完我立刻 X" | 看完这个我立刻删了某个 app |
| 反问 | "你以为是 X？其实是 Y" | 你以为是天赋？其实是熬出来的 |
| 召唤 | "请你立刻 X" / "求求姐妹 X" | 请你立刻把这 3 件事做完 |
| 代入 | "如果我是 X 我会 Y" / "X 个人都该 X 一次" | 如果我是 25 岁，我会立刻搞副业 |
| 承诺 | "X 天搞定 Y" / "X 就能 Y" | 21 天搞定一份副业 |
| 清单 | "3 个 X 就够了" / "X 的 5 种死法" | 副业的 5 种死法 |

这只是初稿骨架。Phase 2 蒸馏接入后，会从真实热贴中自动归纳新骨架。

### 5.3 句式库表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| pattern_id | 文本 | 句式 ID（主键） |
| skeleton | 文本 | 骨架本体，如 "不是 X，是 Y" |
| category | 单选 | 对比/递进/悬念/反问/召唤/代入/承诺/清单 |
| examples | 文本 | 典型例句（从真实热贴抽取，最多 3 条） |
| frequency | 数字 | 在历次采集中出现次数 |
| first_seen / last_seen | 日期 | 首次/最近出现日期 |
| trend | 单选 | 上升/平稳/下降/新句式 |
| recommended_categories | 多选 | 推荐搭配的金词功能位（如"承诺型"句式常配 number + do） |
| user_note | 文本 | 博主人工批注 |

## 7. Agent 核心逻辑：金词蒸馏

### 7.1 蒸馏流程（v2 改写）

Agent 拿到一批已识图的热贴后，执行以下分析：

**第一步：分别提取 title 与 cover_text**
对每条热贴，分两路处理：
- **标题路**：标题 + 完整正文（语境补充）
- **封面路**：封面文案输出结果（独立处理）

**第二步：按功能位识别金词**
对每条候选词，LLM 判断：
- 是否归属 8 个功能位之一（不归属则丢弃，vibe_score < 3）
- 给出 `category`、`source_field`、`vibe_score`
- 抽取 `aliases`（同义变体）

**第三步：识别匹配的句式骨架**
对每条热贴的标题与封面，匹配到 §5.2 的句式骨架；如果发现新骨架（未在句式库），加入候选句式。

**第四步：金词筛选**
候选词需满足：
- vibe_score ≥ 4 （金词味门槛）
- 出现在 2 条以上热贴中，**或** 来源帖互动数据高于本批次中位数
- 在现有金词库中不存在（新鲜度），或趋势为"上升/平稳"

**第五步：趋势比对**
与历史金词库/句式库比对，打"新词/上升/平稳/下降"标签。

### 7.2 蒸馏 prompt 输出结构

```json
{
  "gold_words": [
    {
      "word": "弄脏双手",
      "aliases": ["把手弄脏", "脏手干活"],
      "category": "do",
      "source_field": "title",
      "vibe_score": 8,
      "frequency": 3,
      "related_post_ids": ["xxx", "yyy"],
      "vibe_reason": "命令式动词+具象动作，画面感强"
    }
  ],
  "patterns": [
    {
      "skeleton": "不是 X，是 Y",
      "category": "递进",
      "examples": ["不是想躺平，是太累了"],
      "from_post_ids": ["xxx"]
    }
  ]
}
```

## 8. 周报与简报（v2 核心增量）

### 8.1 实时简报（保留 v1 能力）

每次 /harvest 完成后，CLI 输出本次采集摘要：

```
═══════════════════════════════════════
  本次采集 | 2026-05-16 23:00
═══════════════════════════════════════
📊 采集 187 条 | API $0.32 | 解析 92%

📈 新金词（按 vibe_score 排序）
  1. "情绪股权"  [feel|title|vibe=9]  频次:7
  2. "反脆弱体质" [twist|cover|vibe=8]  频次:5
  3. "认知带宽"  [picture|both|vibe=8] 频次:4

🆕 新句式
  · "X 是当代 Y" — 例：班味是当代瘟疫

🔥 持续升温 / 😴 下降趋势：详见 /report --weekly
═══════════════════════════════════════
```

### 8.2 周报（v2 新增）

**触发**：`/report --weekly` 或周日自动生成

**结构**：

```markdown
# 金词蒸馏周报 | 2026-W20（5-12 ~ 5-18）

## 1. 数据概览
- 采集：428 条热贴（4 次 /harvest）
- 新金词：23 个 | 新句式：4 个
- 互动 TOP3 帖：[链接列表]

## 2. 金词层次分布（按功能位）
| 功能位 | 本周新增 | 累计 | 周环比 |
|--------|---------|------|--------|
| 量化锚 | 8 | 56 | ↑ 30% |
| 身份标签 | 5 | 41 | ↑ 12% |
| 反常识 | 3 | 18 | ↑ 15% |
| ... | ... | ... | ... |

## 3. TOP 金词推荐（按 vibe_score 排序）
按"标题用"和"封面用"分别列出 Top 5。

## 4. 新句式发现
本周从热贴中归纳出的新骨架，配合典型例句。

## 5. 趋势变化
- 升温词：xxx（频次 6→12）
- 降温词：xxx（频次 8→2，可能可标"过气"）

## 6. 选题建议（基于本周原料）
agent 自动从"高 vibe 金词 + 升温句式"组合生成 5 个候选标题/封面。
```
